üî∞ Introduction to Cryptography

What is Cryptography?

Cryptography is the art and science of securing information by transforming it into an unreadable format. It involves creating codes and ciphers to protect data from unauthorized access, ensuring confidentiality, integrity, and authenticity in digital communications.

Classical vs Modern Cryptography

Classical cryptography uses simple mathematical operations and manual techniques, while modern cryptography employs complex algorithms, computer processing, and mathematical theories to provide robust security for digital systems.

‚ö° Modern Cryptographic Tools

RSA (Public-key / Asymmetric)

What: An algorithm where two keys are used ‚Äî a public key to encrypt or verify, and a private key to decrypt or sign.

How: Choose two large random primes p and q, compute n = p*q and œÜ(n)=(p-1)(q-1). Choose public exponent e, compute d as the modular inverse of e modulo œÜ(n).

History: Published in 1977 by Rivest, Shamir, and Adleman.

Diffie‚ÄìHellman Key Exchange

What: A method for two parties to establish a shared secret over an insecure channel.

How: Public parameters (prime p and base g). Party A picks secret a and sends A = g^a mod p. Party B picks secret b and sends B = g^b mod p.

History: Introduced in 1976 by Whitfield Diffie and Martin Hellman.

SHA Family (SHA-1, SHA-2, SHA-3)

What: Hash functions for integrity and cryptographic applications.

How: Block-processing with rotations and bitwise operations, producing fixed-size digests.

History: SHA-1 deprecated; SHA-2 (2001) widely used; SHA-3 standardized later.

MD5

What: Hash function producing a 128-bit digest.

History: Published in 1992, widely used for checksums; now deprecated for secure use.

HMAC

What: Hash-based Message Authentication Code using a secret key and hash function.

History: Used in TLS, API signing, and token authentication.

PBKDF2

What: Converts a password into a derived key with computational cost to slow attackers.

History: Standardized in PKCS #5; used for secure password storage.

AES

What: Symmetric block cipher for fixed-size data blocks.

How: Substitution-permutation network, multiple rounds, key sizes 128/192/256 bits.

History: NIST selected in 2001; based on Rijndael cipher.

RC4

What: Stream cipher producing a keystream XORed with plaintext.

History: Designed in 1987, used in WEP/SSL; now deprecated.

Base64 / Base32

What: Encode binary data into readable ASCII.

History: Developed for safe transport over text-based systems.

Cipher Identifier

Frequency Analysis

Analyzes letter frequencies to help break monoalphabetic ciphers.
